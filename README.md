# ⛈️ 北台灣午後對流個案 - 自動化資料封存網頁系統

利用自動化排程與觀測指標偵測台北盆地之午後對流事件。當觀測數據達設定標準時，雲端系統將自動蒐集雷達回波、閃電落雷、地面氣象要素及综觀天氣圖資，進行整合並於網頁端動態發布，以利後續氣象個案研究與長期統計分析。

🔗 **系統展示網頁主頁**：[https://aswyz4260.github.io/Taipei-Convection-Project/](https://aswyz4260.github.io/Taipei-Convection-Project/)

---

## 🧐 系統執行模式與判定機制

使用者可透過 GitHub Actions 介面，依據研究需求選擇以下三種運行模式：

### 模式 1：定時自動執行（常規檢測與深夜資料補齊排程）
0. 每日台灣時間 11：30 執行QPF定量降水預報下載（每日台灣時間 09:52 提前排隊）
🔺 2026/6/26 更新

* **⚡ 今日事件常規檢測：每日台灣時間 19:37 啟動**
* **判定指標（雷達回波演變分析）**：
1. 於設定之北台灣地理網格範圍內（`緯度: 24.75 ~ 25.30°N / 經度: 121.00 ~ 121.80°E`），雷達回波強度 $\ge 40\text{ dBZ}$ 之覆蓋面積必須大於 $10\text{ km}^2$（約合 6 個網格點）。
2. 上述強回波特徵必須**連續維持達 30 分鐘（含）以上**。
3. 此標準參照 Lin et al. 2012 設定


* **執行邏輯**：系統評估達上述標準時，即判定對流個案成立，隨即啟動圖資擷取與網頁發布程序；若未達標準，系統將自動中斷後續下載流程，並啟動QPF目錄清理機制，以節省雲端儲存空間。*(註：此常規排程內建時間防呆機制，若因 GitHub 伺服器延遲導致跨日開機，程式將自動校正並回溯至對流實際發生之正確日期。)*


* **🌙 昨日圖資深夜補齊：每日台灣時間 00:21 啟動**
* **執行邏輯**：自動回溯前一日之觀測紀錄。若前一日傍晚判定對流成立且已建檔，此排程將以 `force_download` 模式再次執行，用以補齊前一日未能完整下載之圖資與最終觀測站數據。



### 模式 2：手動輸入日期判定（歷史個案回溯）

* **適用情境**：研究人員欲驗證過去特定日期（例如：2026年6月14日）是否符合午後對流判定指標。
* **操作流程**：於設定介面輸入 8 碼日期（YYYYMMDD）並將執行模式選擇為 `run_triggered`。系統將載入該日之歷史雷達數據，重複模式 1 之指標檢驗流程，達標時方進行資料封存。

### 模式 3：手動強制下載（無條件資料擷取）

* **適用情境**：特定日期之午後對流雖未達模式 1 之雷達回波強度或面積標準，但具備特殊科學研究價值（如強烈熱島效應、局地海風鋒面觸發等），需強制保留觀測資料。
* **操作流程**：於設定介面輸入日期並將執行模式切換為 `force_download`。系統將跳過所有指標判斷程序，直接下載該日之完整氣象圖資並更新至展示網頁。

## 🗺️ 展示網頁主頁：局地觀測地圖（index.html）

展示主頁提供即時與歷史個案之局地天氣空間分布，點擊導覽列之「📍返回測站地圖」即可檢視。

1. **🔬 地理邊界篩選**：
地圖測站採自定義之北台灣地理座標邊界（Bounding Box）進行篩選：

$$\text{緯度（LAT）: } 24.75^\circ\text{N} \sim 25.30^\circ\text{N}$$
$$\text{經度（LON）: } 120.90^\circ\text{E} \sim 122.00^\circ\text{E}$$

此設定涵蓋整個大台北盆地、桃園北部地形、基隆沿海及鄰近海域之自動觀測站（採用 WGS84 座標系）。

2. **地圖操作**：
從選單點選個案日期後，即可從地圖上選擇測站查看 溫度雨量時序圖 與 風向風速時序圖

## 🌐 個案日誌展示網頁（cases.html）

個案日誌（`cases.html`）用於呈現已個案之觀測圖資，介面區分為左側列表與右側看板：

### 1. 左側個案列表

* 依日期倒序排列展示所有已建檔之對流事件。
* **綠色 [已紀錄]**：代表研究人員已完成該個案之文字分析與審查報告。
* **紅色 [未審查]**：代表僅具備自動化觀測圖資，尚未建立人工報告。點擊可透過超連結前往 GitHub Issues 頁面填寫表單補交。
* ‼️重新提交相同日期之表單將完全覆蓋原有紀錄。

### 2. 右側觀測看板

* **☂ 全台日累積雨量分布圖**
* 展示 「實測總累積雨量 $\leftrightarrow$ 12小時預報 $\leftrightarrow$ 6小時預報」 之對照。每日 19:30 存檔之全台日累積雨量圖（`daily_rainfall_1930.jpg`），提供研究人員對當日降雨核心空間分布之宏觀掌握。為強化模式預報之校驗功能，系統於每日中午 11:30 自動擷取中央氣象署發布之定量降水預報（Quantitative Precipitation Forecast, QPF）數值產品，包含「未來 6 小時」與「未來 12 小時」之局地降雨分布預測，

* **📡 10-18 LST 雷達回波與閃電落雷同步播放器**
* 配置微調拉桿（每 10 分鐘為一幀）。拖動時，**左側之局地雷達回波圖（無地形版）與右側之即時閃電落雷圖將進行同步幀率切換**，利於比對強對流胞與落雷密集區之空間對位。


* **🌧️ 08-18 LST 地面場觀測演變**
* 內嵌「整點滑桿」與「30分鐘滑桿」。
* 展示局地累積雨量風向圖、氣溫分佈圖，此處之雨量數值均已校正為時雨量。


* **📊 氣象要素與綜觀天氣**
* **觀測數據排行榜 Top 5**：雨量排行榜以 19:00 之最終日累積雨量進行排序，氣溫則為全時段之最高溫。
* **綜觀環境圖集**： 08:00 LST（00UTC）之台北站（46692）與彭佳嶼站（46695）熱力探空斜溫圖（Skew-T）、地面天氣圖、850hPa 綜觀流場及 500hPa 高空槽線圖，點擊均可啟動燈箱（Lightbox）放大檢視。

## 📂 儲存目錄結構與資料規範

系統建檔成功後，將於 repository 中建立以日期（YYYYMMDD）命名之子目錄，其資料分類架構如下：

* `radar_archives/`：存放 10:00 至 18:00，每 10 分鐘一幀之局地雷達回波圖與閃電落雷分布圖。
* `sfc_archives/`： 存放 08:00 至 18:30 之地面降雨分布與局地氣溫向量圖。
* `stn_archives/`：存放結構化觀測數據（`stations_summary.json`，含 WGS84 座標、淨時雨量等）與人工審查報告（`review_summary.json`）。
* `cwaFig_archives/`：存放綜觀環境場分析圖、熱力探空圖及 19:30 日累積雨量圖。
* `qpf_images/`：存放每日 11:30 定量降水預報（12hr, 6hr）。
